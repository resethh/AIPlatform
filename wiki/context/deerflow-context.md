# DeerFlow 上下文管理

## 一句话定位
DeerFlow 的多层联动上下文系统，覆盖从单条消息到跨会话长期记忆的全生命周期，核心目标是在有限 Context Window 内最大化信息密度。

## 三层上下文模型

```
┌────────────────────────────────────────────────┐
│  系统提示（System Prompt）                        │
│  角色定义 + 记忆注入 + 技能目录 + 子 Agent 规则   │
├────────────────────────────────────────────────┤
│  对话历史（Messages）                             │
│  当前会话全部轮次（可被 Summarization 压缩）       │
├────────────────────────────────────────────────┤
│  运行时注入（Runtime Injection）                   │
│  上传文件列表 + 图像 base64 + 工具调用修复         │
└────────────────────────────────────────────────┘
```

- **系统提示**：提供长期个性化背景，Session 开始时组装，含记忆注入结果
- **对话历史**：承载当前 Session 推理链，满后触发 Summarization 压缩
- **运行时注入**：处理多模态输入和工具调用异常恢复（如 JSON 修复）

## 三条核心设计原则

1. **关键路径不阻塞**：记忆更新、Token 统计等耗时操作全部异步，不影响用户响应延迟
2. **渐进式加载**：技能/工具按需读取，初始系统提示只包含目录，避免一次性撑满上下文
3. **原子一致性**：checkpointer 状态持久化和记忆写入使用原子操作，防止进程崩溃导致数据损坏

## 与记忆层的交汇

- **Memory Injection**: 系统提示组装时，调 `format_memory_for_injection()` 注入历史记忆
- **Memory Flush**: 对话历史即将压缩前，先触发 `MemoryUpdateQueue` 提取关键信息落库
