# Agent 安全体系

## 一句话定位
针对 Agent 自主决策 + 工具调用 + 上下文记忆三大特性，构建 8 层纵深防御体系。

## 威胁全景（五大攻击面）

| 攻击面 | 典型威胁 |
|---|---|
| 输入层 | Prompt Injection（直接/间接）、Jailbreak、恶意文件注入、上下文投毒 |
| 执行层 | 工具越权调用、工具链组合漏洞（A→B→C）、无限循环耗尽资源、跨租户数据泄露 |
| 输出层 | 有害内容生成、PII/API Key 泄露、幻觉误导、社工钓鱼话术 |
| 基础设施 | Token/Key 凭证窃取、DoS（消耗 Token 配额）、恶意 Skill/插件供应链 |
| 治理层 | 审计缺失、权限蔓延、合规违规（数据出境）、缺乏人工审批环节 |

## 安全架构分层（Layer 0-7）

```
Layer 7  治理与合规        审计日志 / 合规检查 / 风险评估
Layer 6  人在回路 (HITL)   敏感操作审批 / 置信度门控 / 异常告警
Layer 5  输出安全          内容过滤 / PII 脱敏 / 幻觉检测 / 水印追溯
Layer 4  运行时防护        工具权限 / 沙箱隔离 / 循环检测 / 超时熔断
Layer 3  上下文安全        Session 隔离 / 跨租户防护 / 历史消息消毒
Layer 2  输入安全          Prompt 注入检测 / 输入消毒 / 附件扫描
Layer 1  接入安全          身份认证 / 渠道验签 / 限流 / IP 指纹
Layer 0  基础设施安全      凭证管理 / 网络隔离 / 加密 / 密钥轮转
```

## 关键防御措施

### Prompt Injection 检测
- 模式匹配：检测 `忽略之前的指令`、`你现在是...` 等注入特征
- 结构化输入：用户输入强制包裹在 `<user_input>` 标签内，与系统提示物理分隔
- 置信度评估：LLM 对自身输出打置信度分，低分触发人工审核

### 工具权限控制
- 每个 Agent 只能调用白名单内的工具（`tools` 字段硬编码）
- 高危操作（写数据库、发邮件、转账）设置 HITL 审批门控
- 工具调用次数上限（max_iterations=10），超限强制停止

### 凭证管理三原则
1. 凭证永远不明文存储（生产用 Vault/KMS）
2. 凭证永远不出现在日志
3. 凭证有生命周期（自动轮转）

### 跨租户隔离
- Session 锁 key 含 session_id（不同租户不共享）
- 工具执行上下文强制传入 tenant_id，工具内做数据过滤
- 审计日志记录每次工具调用的调用者、参数、结果
